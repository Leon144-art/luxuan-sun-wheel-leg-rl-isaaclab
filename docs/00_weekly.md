在最小测试之后，进行了更大规模的 smoke test：

## Week 2 (2026-06-22 - 2026-06-28)

### 本周重点

本周主要进入 HopperTrex demo 的实际训练和任务调试阶段。相比上周的环境准备，本周重点放在理解机器人控制任务、尝试不同训练路线，并寻找一个可以稳定运行的基础策略。

本周主要围绕固定腿控制展开。固定腿路线训练难度相对低，可以先作为稳定基线；同时也观察到，如果允许腿部参与控制，理论上可能有更高的运动上限，但训练难度和不稳定性也会明显增加。

### 完成的工作

进行了 HopperTrex demo 中 robot asset、task config、train/play 脚本和 RSL-RL 配置的检查。当前任务主要通过速度命令控制机器人，包括前进速度和 yaw 角速度。本周训练重点放在 fixed-leg 版本，即腿部姿态固定，主要通过两个轮子的速度控制实现平衡、直行和转向。

确定并延续了当前较稳定的固定腿姿态：

```text
hip=-69
knee=15
mirror-knee=True
```

训练和播放时保持相同姿态，避免由于初始姿态不一致导致策略表现不稳定。

早期直接训练完整速度命令时，策略容易出现站立不稳、直行偏航、静止漂移和转向不受控等问题。因此本周对 reward 做了拆分和调整：

- 将 x 方向速度跟踪单独处理；
- 增加 y 方向漂移惩罚；
- 增加 yaw 误差和静止漂移相关惩罚；
- 增加直行时左右轮速度差的辅助约束；
- 对零命令和原地转向场景分别加强约束。

这些调整的目标是减少“为了追速度而牺牲姿态和方向稳定性”的情况，让策略先学会稳定直行、静止和绕中心转向。

本周没有直接从一开始训练完整速度范围，而是采用分阶段方式：

```text
直行稳定
-> 加减速时减少偏航
-> 加入小范围 yaw
-> 扩展到更高 x 速度和 yaw
-> 针对静止和原地转向继续 refinement
```

这种方式比直接训练完整命令范围更稳定。当前策略已经可以在 fixed-leg 设置下完成较长 episode，并且在直行和低速转向方面比早期模型稳定很多。

本周后半段开始补充训练记录流程，包括保存训练参数、关键 checkpoint 和训练过程中的指标。由于一开始主要在快速试错，没有及时系统记录所有训练日志，所以前半段结果主要依赖终端输出和 checkpoint 对比。后续训练会尽量保留更完整的参数和指标记录，便于比较不同模型。

### 验证结果

本周比较明确的结果是，fixed-leg 路线已经能够训练出可用的稳定基线。相比最初直接训练完整速度命令的方式，分阶段训练后模型在以下方面有明显改善；同时也留下了一个问题：如果希望策略一步训练到位，而不是依赖较细的课程安排和多次 refinement，是否需要重新设计 command curriculum、reward 权重调度或初始策略引导方式。

- 能够更稳定地保持完整 episode；
- 直行时偏航减少；
- 低速控制和原地转向更稳定；
- 静止漂移问题有所改善；
- 训练过程中的非轮接触和摔倒情况明显减少。

当前仍需要继续观察的是，针对静止和原地转向优化后，是否会削弱高速前进能力。因此后续可能需要保留不同用途的 checkpoint，分别用于高速行走和低速/原地控制对比。

### 问题和不确定性

当前主要问题包括：

- 静止时仍可能出现轻微漂移或自转；
- 原地转向需要继续确认是否真正绕机器人中心旋转；
- 低速稳定性和高速运动能力之间可能存在权衡；
- 允许腿部参与控制的路线可能上限更高，但训练难度更大，目前还没有形成稳定方案。

### 下周计划

下周计划继续评估最新 checkpoint 的实际表现，重点测试：

- 零命令下是否能稳定保持静止；
- yaw-only 命令下是否能绕中心转；
- 低速前进加转向是否稳定；
- 高速前进能力是否因为低速 refinement 出现退化。

在评估结果明确后，再决定下一步是继续扩展 fixed-leg 模型的速度范围，还是尝试允许腿部参与控制的路线，尝试提高整体运动能力上限。

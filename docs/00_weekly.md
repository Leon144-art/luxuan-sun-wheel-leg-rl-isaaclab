## Week 2 (2026-06-22 - 2026-06-28)

### 本周重点

本周主要进入 HopperTrex demo 的实际训练和任务调试阶段。相比上周的环境准备，本周重点放在理解机器人控制任务、尝试不同训练路线，并寻找一个可以稳定运行的基础策略。

本周主要围绕固定腿控制展开。固定腿路线训练难度相对低，可以先作为稳定基线；同时也观察到，如果允许腿部参与控制，理论上可能有更高的运动上限，但训练难度和不稳定性也会明显增加。

### 完成的工作

进行了 HopperTrex demo 中 robot asset、task config、train/play 脚本和 RSL-RL 配置的检查。当前任务主要通过速度命令控制机器人，包括前进速度和 yaw 角速度。本周训练重点放在 fixed-leg 版本，即腿部姿态固定，主要通过两个轮子的速度控制实现平衡、直行和转向。

确定并延续了当前较稳定的固定腿姿态：

```text
hip=-69
knee=15
mirror-knee=True
```

训练和播放时保持相同姿态，避免由于初始姿态不一致导致策略表现不稳定。

早期直接训练完整速度命令时，策略容易出现站立不稳、直行偏航、静止漂移和转向不受控等问题。因此本周对 reward 做了拆分和调整：

- 将 x 方向速度跟踪单独处理；
- 增加 y 方向漂移惩罚；
- 增加 yaw 误差和静止漂移相关惩罚；
- 增加直行时左右轮速度差的辅助约束；
- 对零命令和原地转向场景分别加强约束。

这些调整的目标是减少“为了追速度而牺牲姿态和方向稳定性”的情况，让策略先学会稳定直行、静止和绕中心转向。

本周没有直接从一开始训练完整速度范围，而是采用分阶段方式：

```text
直行稳定
-> 加减速时减少偏航
-> 加入小范围 yaw
-> 扩展到更高 x 速度和 yaw
-> 针对静止和原地转向继续 refinement
```

这种方式比直接训练完整命令范围更稳定。当前策略已经可以在 fixed-leg 设置下完成较长 episode，并且在直行和低速转向方面比早期模型稳定很多。

本周后半段开始补充训练记录流程，包括保存训练参数、关键 checkpoint 和训练过程中的指标。由于一开始主要在快速试错，没有及时系统记录所有训练日志，所以前半段结果主要依赖终端输出和 checkpoint 对比。后续训练会尽量保留更完整的参数和指标记录，便于比较不同模型。

### 验证结果

本周比较明确的结果是，fixed-leg 路线已经能够训练出可用的稳定基线。相比最初直接训练完整速度命令的方式，分阶段训练后模型在以下方面有明显改善；同时也留下了一个问题：如果希望策略一步训练到位，而不是依赖较细的课程安排和多次 refinement，是否需要重新设计 command curriculum、reward 权重调度或初始策略引导方式。

- 能够更稳定地保持完整 episode；
- 直行时偏航减少；
- 低速控制和原地转向更稳定；
- 静止漂移问题有所改善；
- 训练过程中的非轮接触和摔倒情况明显减少。

当前仍需要继续观察的是，针对静止和原地转向优化后，是否会削弱高速前进能力。因此后续可能需要保留不同用途的 checkpoint，分别用于高速行走和低速/原地控制对比。

### 问题和不确定性

当前主要问题包括：

- 静止时仍可能出现轻微漂移或自转；
- 原地转向需要继续确认是否真正绕机器人中心旋转；
- 低速稳定性和高速运动能力之间可能存在权衡；
- 允许腿部参与控制的路线可能上限更高，但训练难度更大，目前还没有形成稳定方案。

### 下周计划

下周计划继续评估最新 checkpoint 的实际表现，重点测试：

- 零命令下是否能稳定保持静止；
- yaw-only 命令下是否能绕中心转；
- 低速前进加转向是否稳定；
- 高速前进能力是否因为低速 refinement 出现退化。

在评估结果明确后，再决定下一步是继续扩展 fixed-leg 模型的速度范围，还是尝试允许腿部参与控制的路线，尝试提高整体运动能力上限。

## Week 3 (2026-06-29 - 2026-07-05)

### 本周重点

本周主要从 fixed-leg 稳定基线继续推进到更复杂的 wheel-leg / leg-enabled 训练尝试。这部分更多是基于现有基线做的个人探索，还没有形成成熟方案。回头看，推进节奏可能有些偏急，在诊断和对照还不够充分时就引入了更多自由度和训练设计变化，也让后续排查成本变高。相比固定腿版本，轮腿联合控制需要同时处理轮子速度、腿部姿态、机体平衡和接触约束，训练过程明显更不稳定。

同时，本周补充学习了一些强化学习和 policy network 的基础知识，包括 MLP、actor-critic、observation / action / reward 的关系，以及 PPO 训练循环中 rollout、policy update 和 episode termination 的基本含义。这些内容主要用于理解训练脚本、网络结构和日志指标，不是单独的理论推导。

### 完成的工作

在 fixed-leg 结果作为基线的基础上，开始尝试允许腿部参与控制的任务路线。工作重点包括理解腿部 action 对机器人姿态的影响、观察不同 reward 和 curriculum 设置下的训练表现，并尝试让策略先学会更可靠的站立和平衡，再逐步扩展到速度跟踪。

本周也改进了训练观察方式。除了只看 mean reward 和 episode length，也开始更系统地关注 termination、接触惩罚、姿态稳定性、速度跟踪误差和动作变化等指标。这样可以更快判断模型是因为接触终止、root height 过低、姿态翻倒，还是因为速度跟踪失败而表现变差。

此外，对训练记录和命令管理做了进一步整理。相比早期快速试错，本周更重视记录训练命令、保存 checkpoint、保留关键日志，并通过更紧凑的训练控制台观察主要指标，减少只凭单个 reward 数值判断训练质量的情况。

### 验证结果

本周比较明确的结果是，fixed-leg 路线仍然是当前较可靠的稳定基线。轮腿联合控制路线虽然暴露出更高的运动潜力，但还没有形成稳定、可复现的完整策略。

当前训练中已经可以观察到一些局部进展，例如部分设置下模型能够维持一段时间的站立或低速运动，也能在某些阶段改善姿态或速度误差。但整体表现仍不稳定，不同失败模式之间相互耦合。例如，站立质量、速度跟踪、非轮接触、root too low 和动作平滑性会同时影响 episode length，使得单次训练结果很难直接解释。

本周的另一个收获是，对训练日志的理解更加具体。reward、termination 和 observation 的设计会共同决定策略能否分辨哪些动作是有用的，哪些动作只是短期提高 reward 但破坏稳定性。

### 问题和不确定性

目前最大的困境是，wheel-leg 训练的失败原因不容易拆开。模型有时看起来学到了一部分行为，但换一个设置后又会出现接触、压低、摔倒或漂移等问题。reward 或 curriculum 的改动也不总是产生直观结果，有时一个看似合理的放宽或约束，会改变训练早期的失败模式。

当前仍不确定的问题包括：

- 腿部 action 应该在训练早期承担多少自由度；
- 站立稳定、姿态约束和速度跟踪之间如何排序；
- termination 触发过早时，策略是否还能获得足够清晰的学习信号；
- 当前 reward shaping 是否足以引导模型从站立过渡到运动；
- 长时间训练失败时，是模型能力不足、任务设计不合适，还是实验设置没有对齐。

这些问题使得本周的进展不像 fixed-leg 阶段那样直接。虽然还没有得到理想的轮腿策略，但对训练系统的失败模式和日志解释有了更清楚的认识。

### 下周计划

下周计划先静下心来定位问题，亦或是向他人求助，并减少长时间盲目训练，优先做更短、更明确的诊断实验。

同时会继续保留 fixed-leg 作为稳定对照。总之，实际一些，慢慢来吧。（本人已老实）

## Week 5 (2026-07-13 - 2026-07-19)

### 本周重点

本周继续完善 wheel-leg 控制路线，已经形成较可用的站立、直行和 yaw 控制基线。同时开始为实机测试准备独立的 fixed-leg sim-to-real 训练路线，计划先验证只有轮子的车辆，再逐步扩展到轮腿控制。

### 完成的工作

wheel-leg 策略目前能够利用腿部动作补偿底盘倾斜，并完成当前训练范围内的 x / yaw 控制；可调高度尚未进入训练。

新建了面向实机的 fixed-leg 任务。该路线只输出左右轮控制，actor 主要使用 IMU、编码器和轮式里程计等实机可获得的信息。训练中加入了摩擦、重心、编码器误差、控制延迟和执行器参数随机化，并对电机扭矩饱和、持续过载和热衰减进行了近似建模，同时将相关电机状态提供给策略。

训练课程进一步拆分为站立、纯直行、纯 yaw 和混合运动，使不同控制目标可以分别训练和检查。此外，整理了 MJCF、URDF、mesh、固定腿姿态和 yaw-center 坐标说明，为后续协作和实机工作做准备。

### 验证结果

wheel-leg 当前策略在仿真中已经具备较稳定的站立、直行和转向能力，是目前较完整的运动基线。fixed-leg sim-to-real 路线也能够学到稳定站立和直线跟踪，部分模型在零命令下可以使轮子基本保持静止。

训练同时暴露出速度跟踪、电机保护、静止漂移、直行偏航和绕中心转向之间的权衡。这说明面向实机的训练不仅需要追求运动能力，也需要明确执行器限制和不同场景下的控制目标。

### 问题和不确定性

当前电机过载和热模型仍是保守近似，尚未根据真实电机曲线和测试数据标定。仿真中的重心、惯量、摩擦和执行器响应与实车也可能存在差异，因此仿真表现较好并不代表策略可以直接部署，实机命令限制和安全保护仍需要单独实现。

wheel-leg 的可调高度仍是下一阶段功能。现有策略能够通过腿部动作维持底盘姿态，但这不能等同于已经具备高度控制能力。

### 下周计划

下周计划继续改善 fixed-leg 的零命令漂移、纯直行偏航和 yaw-center 表现，并选择一个稳定 checkpoint 作为首轮实机基线。同时准备部署侧的命令裁剪、安全限制和观测接口，优先开展只有轮子的低速实机测试，采集电机、IMU 和编码器数据，再根据实际结果修正仿真参数。

在 fixed-leg 实机路线得到基本验证后，再逐步进入 wheel-leg 部署和可调高度控制。
